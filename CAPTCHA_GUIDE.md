我已经为你成功集成了人机验证功能到 WebUI 中。

### 📌 需要验证码的接口

根据 API 文档，以下接口需要人机验证：

1. **获取邮件注册验证码** - `/public/register/emailCode`
2. **密码登录** - `/public/login`  
3. **签到** - `/auth/user/sign`
4. **重置访问密钥** - `/auth/user/tokenReset`
5. **CDK 兑换** - 暂未实现

### 🔧 使用方法

#### 1. 登录流程
1. 输入邮箱和密码
2. 点击"获取登录验证码"按钮
3. 在新窗口中完成人机验证
4. 复制页面中显示的验证码
5. 将验证码粘贴到"验证码"输入框
6. 点击"登录"按钮

#### 2. 注册流程
1. 输入邮箱和密码
2. 点击"获取注册验证码"按钮
3. 在新窗口中完成人机验证
4. 复制页面中显示的验证码
5. 将验证码粘贴到"验证码"输入框
6. 点击"发送注册验证邮件"
7. 检查邮箱获取邮件验证码
8. 将邮件验证码粘贴到"邮件验证码"输入框
9. 点击"注册"按钮

#### 3. 签到流程
1. 确保已登录
2. 点击"获取签到验证码"按钮
3. 在新窗口中完成人机验证
4. 复制页面中显示的验证码
5. 将验证码粘贴到"验证码"输入框
6. 点击"执行签到"按钮

### 🚀 技术实现细节

#### 客户端配置
- **客户端名称**: `WebUI`
- **验证码URL**: `https://www.mefrp.com/3rdparty/captcha?client=WebUI`

#### 验证码处理流程
1. 用户点击按钮打开验证页面
2. 在新窗口中完成验证
3. 获取 Base64 编码的验证字符串
4. 解码后格式为 `token||client`
5. 将 `token` 部分作为 `captchaToken` 传递给API

#### 后端处理
后端代码已经更新，正确处理验证码参数：

```python
# 登录示例
client.login(username, pwd, captcha_token)

# 获取注册邮件验证码示例  
client.get_register_email_code(email, captcha_token)

# 签到示例
client.sign(captcha_token)
```

### ⚠️ 重要警告

1. **不要尝试用验证码访问其他接口** - 这会导致IP被封禁
2. **每个验证码只能使用一次**
3. **验证码有时效性**，建议立即使用
4. **客户端名称不要使用中文**
