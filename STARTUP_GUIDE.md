# MEFrp WebUI 启动指南

## ⚠️ 关于 WARNING 警告

当您看到以下警告时：
```
WARNING: This is a development server. Do not use it in production deployment.
```

**这是正常的！不是错误！**

- ✅ WebUI 可以正常运行
- ✅ 所有功能正常工作
- ⚠️ 只是提醒不要用于商业生产环境

## 🚀 启动方式

### 方式一：开发模式（默认，推荐）

**文件：** `launch.bat`

**特点：**
- 🚀 启动快速
- 💻 适合开发和测试
- 🖥️ 会显示开发服务器警告

**使用方法：**
1. 双击 `launch.bat`
2. 等待启动完成
3. 访问 http://127.0.0.1:5000

### 方式二：生产模式（稳定）

**文件：** `launch_production.bat`

**特点：**
- 🛡️ 更稳定、安全
- 🔧 使用专业的 WSGI 服务器
- ⚠️ 首次启动需要安装 waitress

**使用方法：**
1. 双击 `launch_production.bat`
2. 等待自动安装 waitress（仅首次）
3. 访问 http://127.0.0.1:5000
4. **没有警告提示！**

### 方式三：手动启动

**命令：**
```batch
cd e:\桌面\1
python_portable\python.exe app.py
```

## 🔧 依赖说明

启动脚本会自动检查和安装以下依赖：

- ✅ Python 便携版
- ✅ Flask（Web 框架）
- ✅ Requests（HTTP 库）
- ✅ MEFrpLib（MEFrp API 库）

**生产模式额外安装：**
- ✅ Waitress（WSGI 服务器）

## 📝 常见问题

### Q1: 为什么开发模式会有警告？

**A:** Flask 自带的开发服务器仅用于开发测试，生产环境应该使用专业的 WSGI 服务器。这是 Flask 官方推荐的做法。

### Q2: 生产模式有什么优势？

**A:** 
- 更稳定，不容易崩溃
- 支持多线程/多进程并发
- 更好的错误处理
- 没有开发警告

### Q3: 警告会影响使用吗？

**A:** 完全不会！这只是提醒，WebUI 可以正常工作。

### Q4: 如何选择启动方式？

**A:**
- 日常使用 → `launch_production.bat`
- 开发调试 → `launch.bat`
- 自定义配置 → 手动启动

## 🛠️ 故障排除

### 问题：启动失败

**解决方案：**
1. 确保已运行 `start.bat` 安装 Python
2. 检查依赖是否完整
3. 查看错误提示信息
4. 尝试重新安装依赖

### 问题：端口被占用

**解决方案：**
1. 修改 `app.py` 中的端口号
2. 或关闭占用 5000 端口的程序

### 问题：无法安装依赖

**解决方案：**
1. 检查网络连接
2. 使用管理员权限运行
3. 手动安装：`pip install flask requests MEFrpLib`
