# ⚠️ 签到验证码使用说明

## 重要提示

签到功能**必须**完成人机验证才能成功！

## 验证码地址

**正确的验证码地址**：
```
https://www.mefrp.com/3rdparty/captcha?client=CODENAME
```

## 获取验证码流程

### Step 1: 点击"打开验证码"按钮

在签到区域，点击 **"打开验证码"** 按钮。

### Step 2: 完成人机验证

1. 浏览器会弹出新窗口
2. 访问 `https://www.mefrp.com/3rdparty/captcha?client=CODENAME`
3. 完成验证挑战（可能是：
   - 滑动验证（拖动滑块到正确位置）
   - 点选验证（点击正确的图片）
   - 拼图验证等）

### Step 3: 获取验证码 Token

验证成功后，页面会显示一个 **Token** 或 **验证码**。

**可能的形式**：
- 页面显示一串字符，如：`abc123def456`
- 页面自动复制到剪贴板
- 页面显示"验证成功"

### Step 4: 复制验证码

1. **选中**验证码字符（如果有显示）
2. **复制**到剪贴板（Ctrl + C）
3. **粘贴**到签到区域的验证码输入框中

### Step 5: 提交签到

点击 **"确认签到"** 按钮完成签到。

## 常见问题

### ❌ 错误：人机验证失败

**原因**：
- 验证码已过期
- 验证码已使用（每个只能用一次）
- 未正确填写验证码
- 网络问题导致验证失败

**解决方案**：
1. 重新点击"打开验证码"
2. 重新完成验证
3. 立即复制新的验证码
4. 粘贴到输入框
5. 立即点击"确认签到"

### ❌ 错误：验证码为空

**原因**：
- 未完成验证就复制
- 验证未成功就关闭窗口

**解决方案**：
1. 确保完成验证
2. 确保页面显示验证成功
3. 复制验证码

### ❌ 验证窗口无法打开

**原因**：
- 浏览器阻止弹出窗口
- 网络问题

**解决方案**：
1. 允许浏览器弹出窗口
2. 检查网络连接
3. 手动访问验证码地址

### ❌ 验证码无法复制

**原因**：
- 验证码可能需要手动选择
- 页面可能有复制限制

**解决方案**：
1. 手动选中验证码文本
2. 右键选择"复制"
3. 或使用 Ctrl + A 全选，然后 Ctrl + C

## 错误处理改进

现在系统已经改进了错误处理：

**之前**：
- 直接显示 `{"error": "人机验证失败"}`
- 不清楚下一步该怎么做

**现在**：
- 如果需要验证码，会显示友好提示：
  ```json
  {
    "error": "签到需要验证码",
    "message": "请先获取验证码，填写后再签到",
    "need_captcha": true
  }
  ```
- 界面会显示通知："需要验证码：请先获取验证码，填写后再签到"

## 注意事项

### ⏰ 时效性

- 验证码**只能用一次**
- 验证码有**时效限制**（通常几分钟内有效）
- 建议**立即使用**，不要等待

### 🔄 频繁获取

- **不要频繁**获取验证码
- 可能触发**风控机制**
- 如果被风控，需要等待一段时间后再试

### 🔐 安全性

- 验证码是**一次性**的安全令牌
- 不会泄露您的账户信息
- 只是证明您是真人而非机器人

### 💡 最佳实践

1. **准备好所有信息**再开始验证
2. **快速完成**验证流程
3. **立即复制**验证码
4. **立即提交**签到请求
5. 如果失败，**重新开始**整个流程

## 验证码格式

验证码通常是：
- **长度**：10-50 个字符
- **格式**：字母 + 数字的组合
- **示例**：`a1b2c3d4e5f6g7h8`

## 技术说明

### API 调用

签到 API 会将验证码发送到 MEFrp 服务器验证：

```javascript
// 发送签到请求
fetch('/api/sign', {
    method: 'POST',
    headers: {'Content-Type': 'application/json'},
    body: JSON.stringify({
        captcha_token: '验证码内容'
    })
})
```

### 服务器验证

MEFrp 服务器会：
1. 接收验证码 Token
2. 验证 Token 是否有效
3. 检查 Token 是否已使用
4. 检查 Token 是否在有效期内
5. 如果都通过，执行签到逻辑
6. 返回签到结果

## 调试技巧

### 查看浏览器控制台

1. 按 **F12** 打开开发者工具
2. 切换到 **Console** 标签
3. 完成验证和签到
4. 查看是否有错误信息

### 检查网络请求

1. 切换到 **Network** 标签
2. 筛选 **fetch** 类型
3. 查看 `/api/sign` 请求
4. 查看请求和响应详情

### 清除缓存

如果验证码一直无效，尝试：
1. 清除浏览器缓存
2. 使用无痕/隐私模式
3. 尝试其他浏览器

## 获取帮助

如果问题仍然存在：

1. **确认验证码格式**：验证码应该是有效的 Token
2. **检查网络**：确保网络连接正常
3. **尝试其他浏览器**：排除浏览器兼容性问题
4. **等待一段时间**：如果被风控，需要等待
5. **联系官方**：如果持续失败，联系 MEFrp 客服
